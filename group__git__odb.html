Enumeration Type Documentation

enum git_otype

Basic type (loose or packed) of any Git object.

Enumerator:
GIT_OBJ_ANY 

Object can be any of the following.

GIT_OBJ_BAD 

Object is invalid.

GIT_OBJ__EXT1 

Reserved for future use.

GIT_OBJ_COMMIT 

A commit object.

GIT_OBJ_TREE 

A tree (directory listing) object.

GIT_OBJ_BLOB 

A file revision object.

GIT_OBJ_TAG 

An annotated tag object.

GIT_OBJ__EXT2 

Reserved for future use.

GIT_OBJ_OFS_DELTA 

A delta, base is given by an offset.

GIT_OBJ_REF_DELTA 

A delta, base is given by object id.

int git_odb_read_header ( git_rawobj *  out,
git_odb *  db,
const git_oid *  id 
)

Read the header of an object from the database, without reading its full contents.

Only the 'type' and 'len' fields of the git_rawobj structure are filled. The 'data' pointer will always be NULL.

The raw object pointed by 'out' doesn't need to be manually closed with git_obj_close().

Parameters:
out object descriptor to populate upon reading.
db database to search for the object in.
id identity of the object to read.
Returns:
  • GIT_SUCCESS if the object was read;
  • GIT_ENOTFOUND if the object is not in the database.
